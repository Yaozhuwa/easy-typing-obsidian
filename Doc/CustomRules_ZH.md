# 规则引擎

Easy Typing 使用统一的**规则引擎**处理所有文本转换——从内置的全角半角标点转换，到你自己编写的自定义快捷输入规则，都由同一个引擎驱动。

## 内置规则速览

插件自带一组内置规则，覆盖常见的中日韩编辑场景。这些规则可以在**内置规则**设置标签页中查看和切换。

### 自动配对（优先级 10）

输入中日韩开括号或引号时，自动插入对应的闭合符号：
- `【` → `【|】`、`（` → `（|）`、`《` → `《|》`
- `"` → `"|"`、`'` → `'|'`、`「` → `「|」`、`『` → `『|』`

> `|` 表示光标位置。如果光标右侧已存在闭合符号，输入时会跳过而不是重复插入。

删除中日韩开括号时，会同时删除配对的闭合括号。

### 全角 ↔ 半角转换

**半角转全角**（优先级 15）：在中日韩字符后，英文标点自动转换为对应的全角标点：
- `,` → `，`、`.` → `。`、`?` → `？`、`!` → `！`
- `:` → `：`、`;` → `；`、`(` → `（|）`

**全角转半角**（优先级 3）：连续输入两个相同的全角标点，转换为半角：
- `。。` → `.`、`！！` → `!`、`？？` → `?`
- `，，` → `,`、`：：` → `:`、`；；` → `;`
- `（（` → `()`、`《《` → `<`

### 符号转换（优先级 10）

| 输入 | 输出 | 说明 |
|------|------|------|
| `··`（两个中文间隔号） | `` `|` `` | 行内代码 |
| `` `· `` | 围栏代码块 | 代码块 |
| `￥￥` / `¥¥` / `$￥` / `$¥` | `$|$` | 行内公式 |
| `$$`（行尾） | `$$\n|\n$$` | 块级公式 |
| 行首 `》` | `> ` | 引用块 |
| 行首 `、` | `/` | 快捷输入 |

### 快速删除配对（优先级 30）

按退格键快速删除配对结构：

| 操作 | 效果 |
|------|------|
| 在 `$|$` 中按退格 | 同时删除两个 `$` |
| 在 `==|==` 中按退格 | 同时删除两个 `==` |
| 在 `$$\n|\n$$` 中按退格 | 删除整个块级公式 |
| 在空代码块中按退格 | 删除整个代码块 |
| 在 `[[链接]]` 或 `![[嵌入]]` 中按退格 | 删除整个 wikilink |

### 选中文字包裹（优先级 40）

选中文字后，输入触发字符进行包裹：

| 触发符 | 输出 |
|--------|------|
| `【` | `[选中文字]` |
| `￥` / `¥` | `$选中文字$` |
| `"` / `"` | `"选中文字"` / `"选中文字"` |
| `'` / `'` | `'选中文字'` / `'选中文字'` |
| `《` | `《选中文字》` |
| `（` | `（选中文字）` |

### 引用处理（优先级 50）

- 在引用块上下文中输入 `>` 或 `》` 会添加一级引用前缀 `> `
- 自动在 `>` 前缀后补空格

## 自定义规则快速上手

只需 3 步即可创建你的第一条规则：

### 第 1 步：打开设置

打开插件设置 → **用户规则** 标签页 → 点击 `+` 按钮。

### 第 2 步：填写规则

以"输入 `->` 自动变为 `→`"为例：

| 面板字段 | 填入 |
|----------|------|
| 类型 | 输入 |
| **光标前匹配** | `->` |
| **替换内容** | `→$0` |

其中 `$0` 表示替换后光标停留的位置。其他字段保持默认即可。

### 第 3 步：测试

在编辑器中输入 `->` ——会自动变为 `→`，光标停在箭头后面。

> 想了解更多功能？继续阅读下面的完整参考。

## 规则编辑面板

点击 `+`（添加）或齿轮图标（编辑）会打开规则编辑面板。面板分为以下几个区域：

### 基本设置

| 字段 | 说明 |
|------|------|
| **类型** | 三种规则类型，决定规则何时触发（见下文） |
| **触发方式** | 自动触发还是按 Tab 触发（仅输入规则可见） |

**三种规则类型**：

| 类型 | 触发时机 | 典型用途 |
|------|---------|---------|
| **输入** | 输入文字时自动触发 | 文本替换、符号转换、代码片段 |
| **删除** | 按退格键时触发 | 删除配对符号 |
| **选中替换** | 选中文字后输入字符时触发 | 给选中文字加包裹符号 |

**两种触发方式**（仅输入规则）：

| 模式 | 说明 |
|------|------|
| **自动** | 输入时模式匹配后立即触发（默认） |
| **Tab** | 输入模式后需按 Tab 才触发，避免正常输入时意外展开 |

### 匹配条件

| 字段 | 说明 |
|------|------|
| **光标前匹配** | 光标**左侧**需要匹配的文本。选中替换类型下为"触发按键字符" |
| **光标后匹配** | 光标**右侧**需要匹配的文本（可选，选中替换类型不可用） |
| **使用正则表达式匹配** | 开启后，光标前/后匹配字段将作为正则表达式解析 |
| **正则标志** | 仅在开启正则匹配时可见，目前支持 `i` / `m` / `u` |

**关于匹配方式**：
- 默认为**纯文本匹配**——你填什么，就精确匹配什么
- 开启"使用正则表达式匹配"后，可以使用正则表达式的完整语法（如 `\d+`、`[a-z]+`、`(.*?)` 等）
- 当前支持的正则标志：
  - `i`：大小写不敏感
  - `m`：多行模式（`^` / `$` 匹配行首行尾）
  - `u`：Unicode 模式（支持更完整的 Unicode 字符与 `\p{...}` 属性匹配）
- 规则引擎仍然要求匹配**紧贴光标**：
  - 左侧匹配必须以光标左边结束
  - 右侧匹配必须从光标右边开始
- 选中替换规则的"触发按键字符"字段中，每个字符都是一个独立的触发键（如填 `【￥` 表示 `【` 和 `￥` 都能触发）

### 替换

| 字段 | 说明 |
|------|------|
| **使用函数式替换** | 开启后，替换内容字段变为 JavaScript 代码编辑器 |
| **替换内容** | 匹配成功后用来替换的文本（支持模板语法，见下文） |

### 其他

| 字段 | 说明 |
|------|------|
| **作用域** | 可多选，限定规则生效的上下文 |
| **语言** | 限定到特定代码块语言，如 `python`（只要作用域包含"代码"就可见） |
| **优先级** | 数值越小优先级越高，默认 100 |
| **描述** | 方便自己辨认的备注 |

**作用域选项**：

| 选项 | 含义 |
|------|------|
| **全部** | 任何上下文都生效（默认） |
| **文本** | 仅在普通 Markdown 文本中生效 |
| **公式** | 仅在 `$...$` 或 `$$...$$` 内部生效 |
| **代码** | 仅在围栏代码块内部生效 |

> 可以组合多个作用域，例如“文本 + 代码”。  
> 如果同时设置了“代码语言”，该语言限制只会在代码作用域下生效，不会影响文本或公式作用域的匹配。

## 替换模板语法

替换内容支持以下特殊语法：

### Tabstop 占位符

| 语法 | 含义 |
|------|------|
| `$0` | 替换后的主光标位置 |
| `$1`、`$2`... | 额外的光标位置（用 Tab 依次跳转） |
| `${1:默认文本}` | 带默认文本的占位符（Tab 跳转时文本被选中） |

光标跳转顺序为 $0 → $1 → $2 → ...

### 捕获组引用（需开启正则匹配）

| 语法 | 含义 |
|------|------|
| `[[0]]` | 光标前匹配的整个匹配结果 |
| `[[1]]`、`[[2]]`... | 光标前匹配的第 n 个捕获组 |
| `[[R0]]` | 光标后匹配的整个匹配结果 |
| `[[R1]]`、`[[R2]]`... | 光标后匹配的第 n 个捕获组 |

> `[[n]]` 优先取左侧分组；如果左侧没有该下标，回退到右侧。使用 `[[Rn]]` 显式引用右侧。

### 特殊变量（选中替换规则）

| 语法 | 含义 |
|------|------|
| `${SEL}` | 选中的文本 |
| `${KEY}` | 触发的按键 |

### 转义序列

| 输入 | 输出 |
|------|------|
| `\n` | 换行符 |
| `\t` | Tab |
| `\r` | 回车 |
| `\\` | 字面反斜杠 |

> **注意**：在面板中直接输入 `\n` 就代表换行，不需要写成 `\\n`。但在 JSON 文件中，由于 JSON 本身的转义规则，需要写成 `\\n`。

## 函数替换

开启"使用函数式替换"后，替换内容字段变为一个带 JavaScript 语法高亮的代码编辑器。你在其中编写的代码会被作为函数体执行。

### 参数

| 规则类型 | 参数 | 说明 |
|---------|------|------|
| 输入 / 删除 | `leftMatches: string[]` | `leftMatches[0]` = 整个左侧匹配，`[1]` = 第一个捕获组... |
| 输入 / 删除 | `rightMatches: string[]` | `rightMatches[0]` = 整个右侧匹配... |
| 选中替换 | `selectionText: string` | 选中的文本 |
| 选中替换 | `key: string` | 触发的按键 |

### 返回值

- 返回 `string` → 进入标准模板处理（支持 `$0`、`[[1]]` 等）
- 返回 `undefined` → 跳过此规则（不进行替换）

### 示例

输入 `/date` 插入当前日期：

面板填写方式：

| 字段 | 值 |
|------|-----|
| 光标前匹配 | `/date` |
| 使用函数式替换 | 开启 |
| 替换内容 | 见下方代码 |

```javascript
const d = new Date();
return d.toISOString().slice(0, 10) + '$0';
```

## 优先级系统

规则按优先级顺序求值（数值越小优先级越高），**第一个匹配的规则生效**，后续规则不再求值。

| 优先级范围 | 用途 |
|-----------|------|
| 1–10 | 核心内置规则（自动配对、基础转换） |
| 15 | 半角转全角 |
| 20 | 全角转半角 |
| 30 | 删除配对规则 |
| 40 | 选中包裹规则 |
| 50 | 引用处理 |
| 100+ | 用户自定义规则（默认：100） |

## 管理规则

### 设置面板

设置面板有两个规则标签页：

| 标签页 | 功能 |
|--------|------|
| **内置规则** | 查看和切换内置规则。已删除的内置规则可以恢复。 |
| **用户规则** | 创建、编辑、删除和排序自定义规则。支持拖拽排序。 |

### 导入 / 导出

- **导出**：将所有用户规则下载为 JSON 文件
- **导入**：上传 JSON 文件导入规则。重复的规则（相同的 trigger + trigger_right + options）会自动跳过。

### 规则存储

规则以 JSON 文件存储在插件目录中：
- `builtin-rules.json` — 内置规则（自动生成，可自定义）
- `user-rules.json` — 用户自定义规则

## 实用示例集

### 简单文本替换（输入规则）

**将 `->` 转为 `→`**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 输入 |
| 光标前匹配 | `->` |
| 替换内容 | `→$0` |

**将 `:)` 转为表情**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 输入 |
| 光标前匹配 | `:)` |
| 替换内容 | `😀$0` |

### 正则匹配规则（输入规则）

**行首 `note-call` → Obsidian Callout 块**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 输入 |
| 光标前匹配 | `^([\w-]+)-call` |
| 使用正则表达式匹配 | 开启 |
| 正则标志 | `m` |
| 替换内容 | `> [![[1]]] $0\n> $1` |

**将单独的 `i` 自动改成大写 `I`**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 输入 |
| 光标前匹配 | `(^|[ \t])i([ \t.,!?;:])` |
| 使用正则表达式匹配 | 开启 |
| 正则标志 | `m` |
| 替换内容 | `[[1]]I[[2]]$0` |

> 这个规则会在行首，或输入空格 / Tab / 常见英文标点后触发，把单独的 `i` 改成 `I`，并保留前后的分隔符。

### 删除规则

**任务项退化为列表项**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 删除 |
| 光标前匹配 | `([-+*]) \[.\] ` |
| 使用正则表达式匹配 | 开启 |
| 替换内容 | `[[1]] ` |

### 选中替换规则

**选中文字 + `-` 添加删除线**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 选中替换 |
| 触发按键字符 | `-` |
| 替换内容 | `~~${SEL}~~` |

### Tab 触发规则

**输入 `lorem` 后按 Tab 展开占位文本**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 输入 |
| 触发方式 | Tab |
| 光标前匹配 | `lorem` |
| 替换内容 | `Lorem ipsum dolor sit amet, consectetur adipiscing elit.$0` |

### 函数规则

**智能时间插入**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 输入 |
| 光标前匹配 | `/now` |
| 使用函数式替换 | 开启 |
| 替换内容 | 见下方代码 |

```javascript
const d = new Date();
const pad = n => String(n).padStart(2, '0');
return `${d.getFullYear()}-${pad(d.getMonth()+1)}-${pad(d.getDate())} ${pad(d.getHours())}:${pad(d.getMinutes())}$0`;
```

### 作用域限定规则

**仅在公式中生效的规则**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 输入 |
| 光标前匹配 | `oiint` |
| 替换内容 | `\oiint$0` |
| 作用域 | 公式 |

**仅在 Python 代码块中生效的规则**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 输入 |
| 触发方式 | Tab |
| 光标前匹配 | `ifmain` |
| 替换内容 | `if __name__ == '__main__':\n    $0` |
| 作用域 | 代码 |
| 语言 | `python` |

**文本和 Python 代码块共用的规则**：

| 面板字段 | 值 |
|----------|-----|
| 类型 | 输入 |
| 光标前匹配 | `todo` |
| 替换内容 | `TODO: $0` |
| 作用域 | 文本 + 代码 |
| 语言 | `python` |

> 上面这条规则会在普通文本中生效，也会在 Python 代码块中生效；在其他语言代码块中不会生效。

## 附录：JSON 格式与选项标志

通过导入/导出功能，规则以 JSON 格式保存。了解 JSON 字段与面板设置的对应关系，有助于批量编辑规则。

### JSON 字段对照表

| JSON 字段 | 面板对应 | 说明 |
|-----------|---------|------|
| `trigger` | 光标前匹配 / 触发按键字符 | 匹配模式或触发字符 |
| `trigger_right` | 光标后匹配 | 光标右侧匹配（可选） |
| `replacement` | 替换内容 | 替换模板或函数体 |
| `options` | 多个设置项的编码 | 见下方选项标志表 |
| `priority` | 优先级 | 数值越小越先匹配（默认：100） |
| `enabled` | 开关 | 是否启用（默认：true） |
| `description` | 描述 | 可读描述 |
| `scope_language` | 语言 | 限定到特定代码块语言 |
| `regex_flags` | 正则标志 | 仅正则规则使用，支持 `i` / `m` / `u` |

### 选项标志（`options` 字段）

`options` 是一个字符串，将面板中多个设置编码为单字母标志的组合：

| 标志 | 面板对应 | 含义 |
|------|---------|------|
| `d` | 类型 = 删除 | 删除规则 |
| `s` | 类型 = 选中替换 | 选中替换规则 |
| _(无 d/s)_ | 类型 = 输入 | 输入规则（默认） |
| `r` | 使用正则表达式匹配 = 开启 | trigger 和 trigger_right 为正则表达式 |
| `F` | 使用函数式替换 = 开启 | replacement 为 JavaScript 函数体 |
| `T` | 触发方式 = Tab | 仅按 Tab 时触发 |
| `t` | 作用域 = 文本 | 仅在文本中生效 |
| `f` | 作用域 = 公式 | 仅在公式中生效 |
| `c` | 作用域 = 代码 | 仅在代码中生效 |
| `a` | 作用域 = 全部 | 任何上下文（默认，无需指定） |

多个标志可组合，如 `dr` = 删除 + 正则，`cT` = 代码作用域 + Tab 触发。

### 正则标志（`regex_flags` 字段）

`regex_flags` 是独立字段，不编码进 `options`。当前支持：

| 值 | 含义 |
|----|------|
| `i` | 大小写不敏感 |
| `m` | 多行模式 |
| `u` | Unicode 模式 |
| 组合 | 例如 `im`、`iu` |

### JSON 示例

```json
{
  "trigger": "(?<=^|\\n)([\\w-]+)-call",
  "replacement": "> [![[1]]] $0\\n> $1",
  "options": "r",
  "regex_flags": "m",
  "priority": 100,
  "description": "行首 callout 快捷输入"
}
```

对应面板设置：类型=输入，光标前匹配为正则，正则标志=`m`，替换内容为 callout 模板，优先级=100。

> **面板输入 vs JSON 的区别**：在面板中直接输入 `\n` 代表换行、`\t` 代表 Tab；在 JSON 中由于 JSON 自身的转义规则，换行需要写成 `\\n`，Tab 需要写成 `\\t`。
